# Preview 환경 현장 흐름 스모크 테스트 체크리스트

## 목적

운영 Google Sheets/Drive에 불필요한 데이터가 남지 않도록 Preview 또는 테스트 표본에서만 로그인, 수형 분기, 조건부 입력, 차단 제출 흐름을 확인한다.

## 전제

- 테스트 표본 ID는 `TEST-*` 형식을 사용한다.
- 운영 표본으로 최종 제출 테스트를 하지 않는다.
- 사진 업로드 테스트는 테스트 Drive 폴더 또는 테스트 표본에만 수행한다.
- 실제 조사원 비밀번호 입력과 제출은 민감정보 전송 및 운영 데이터 생성 가능성이 있으므로 실행 직전에 사용자 확인을 받는다.

## 준비 항목

- `sample_master`에 `TEST-APPLE-001` 추가
  - `crop`: 사과
  - `variety`: 후지 또는 홍로
  - `surveyor_id`: S01
  - `status`: pending
- `sample_master`에 `TEST-PEAR-001` 추가
  - `crop`: 배
  - `variety`: 신고
  - `surveyor_id`: S01
  - `status`: pending
- Preview 환경변수는 Production과 동일한 구조로 설정하되, 가능하면 테스트 Sheet/Drive를 사용한다.

## 테스트 시나리오

### 로그인 및 표본 목록

- S01 로그인.
- TEST 사과/배 표본이 S01 목록에 표시되는지 확인.
- admin 로그인 시 TEST 표본 2건이 전체 목록에 표시되는지 확인.

### 사과 수형 분기

- `TEST-APPLE-001` 진입.
- 조사 유형으로 `농가 기본 정보 조사` 선택.
- `재배 수형` 선택지가 사과 목록만 표시되는지 확인.
- `기타` 선택 시 `기타 수형명 또는 설명` 필드 표시 확인.
- `확인불가` 선택 시 `확인불가 사유` 필드 표시 확인.

### 배 수형 분기

- `TEST-PEAR-001` 진입.
- 조사 유형으로 `농가 기본 정보 조사` 선택.
- `재배 수형` 선택지가 배 목록만 표시되는지 확인.
- `기타` 선택 시 `기타 수형명 또는 설명` 필드 표시 확인.
- `확인불가` 선택 시 `확인불가 사유` 필드 표시 확인.

### 특이사항 빠른 입력

- `특이사항 없음`, `수형 혼재`, `병해충 많음`, `일부 수확`, `측정 불가` 버튼이 표시되는지 확인.
- 각 버튼 클릭 시 특이사항 필드에 대응 문장이 입력되는지 확인.

### 제출 차단

- 수형 `기타` 선택 후 설명 미입력 상태로 제출 시 차단 확인.
- 수형 `확인불가` 선택 후 사유 미입력 상태로 제출 시 차단 확인.
- 특이사항 `.` 또는 `모름` 입력 후 제출 시 차단 확인.
- 병해충 `많음` 선택 후 사유 없이 제출 시 차단 확인.
- 일부 수확 `O` 선택 후 수확량 없이 제출 시 차단 확인.
- GPS 미수집 또는 필수 사진 미업로드 상태에서는 최종 제출이 차단되는지 확인.

## 정리

- TEST 제출이 생성된 경우 `survey_submissions`, `survey_answers`, `gps_log`, `media_files`에서 TEST 관련 행을 확인 후 보존 또는 삭제 방침을 기록한다.
- TEST 사진이 생성된 경우 Drive 테스트 폴더에서 파일 ID를 기록하고 정리 방침을 기록한다.
